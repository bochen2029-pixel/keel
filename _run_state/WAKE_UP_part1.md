## 3.5 · The full intent & spirit — the operator's own framing (capture, so no future session re-explains it)

*Added 2026-06-14 from the operator directly, to retire the re-explanation tax permanently — across sessions, instances, and any future reader (human or model). This is the load-bearing **why**. Read it as canon-of-intent; everything KEEL does serves it.*

**Why KEEL exists at all.** Building project after project — NightClerk, NightScribe, photo2deck, the companions, TARS, the games — the operator kept re-building the *same* AI/LLM substrate by hand: the same local vision-LLM, the same Whisper ears, the same privacy filtering, the same cheap-vs-frontier routing, the same memory and tool wiring. Every project reinvented the wheel. KEEL is the decision to **build that common core once, correctly, from first principles — as versatile, flexible, and invariant as it can possibly be — and never rewrite it again.** He could not just adopt OpenCode or Claude Code, because those are *agents you talk to*, not a *substrate you embed*. So KEEL is the substrate: **his own, custom-coded, sovereign — for full flexibility, local-first, online-API-capable, intelligently routing between them.** The custom-coded-by-him part is itself the point: ownership and total flexibility, not buy-vs-build convenience.

**One frozen core, three destinies — differing only by which modules are toggled on.**
1. **An embeddable, reusable AI bundle** dropped into his own apps and games — *the .NET of his AI apps.* As software shops built on .NET instead of each re-writing a framework, KEEL lets him build an LLM-infused app or game without re-coding the AI/LLM piece from scratch. (His Backrooms game already consumed KEEL and shipped — a throwaway proof that the embed path works, **not** the point; do not fixate on it or any single use case.)
2. **His own personal AI harness/assistant** — a sovereign, verifiable, *embeddable* daily-driver he himself coded (an OpenClaw-class assistant, but local-first and grounded by a non-model oracle, not a YOLO cloud agent).
3. **Aspirationally, the orchestration kernel of an intelligence that can run an entire organization** — the same loop, scaled up by slotting in more modules.

**The unifying property is scale-invariance and case-agnosticism.** A single, deliberately *simple* invariant kernel loop + the ten frozen contracts, that **reduces to the minimal case** for a game's AI module and **extends to an org-scale orchestrator** — purely by which modules are slotted, toggled, or swapped, **never by rewriting the loop.** That is the genome / stem-cell; that is the name KEEL (the backbone laid first that everything is built upon and that keeps the vessel upright in any sea). The richness of the assistant or the orchestrator never bloats the core — it lives in toggleable modules and cells. The one discipline that makes all of this real: **the loop and the ten contracts stay frozen and simple; you scale by slotting, never by editing them.** (This dissolves the apparent "minimal-game vs maximal-assistant" tension — it was never a tension; it is what modularity is *for*.)

**The concrete defaults — chosen, not arbitrary (the primitives he kept reaching for).** You cannot abstract all the way down; at some point you pick concrete primitives, and these are the ones his own projects converged on: **Qwen3.5-9B** (native early-fusion vision — brain *and* eyes in one model) as the local tier; **Whisper** as ears; the **OpenAI Privacy Filter + regex** as the privacy layer; **DeepSeek V4 Pro** as cheap-API; **Claude Opus 4.8** as frontier (hence the three-tier routing); **Qwen3-0.6B** as the embedder/reranker; **SQLite** as the index; **MCP** as a first-class tool membrane. The 9B local is deliberate: the stronger Qwen3.6-27B needs an RTX-5090 / 32 GB that is too uncommon to assume as a default, so the 9B is the sovereign workhorse. All are swappable via the substrate resolver + `keel.lock` — defaults, not hardwiring.

**On privacy.** The operator personally does not need it. It is **pre-positioned for an eventual for-others use** — far cheaper to design the seam in now than to retrofit it later. So privacy is forward-design, not a personal-urgency feature.

**On the language (Rust now; C/C++ a contemplated future).** Rust was chosen after a genuine fight (the borrow checker as a non-model I5 oracle on the source — the bug class a 24/7 self can least afford). But for a *true kernel* the operator's instinct leans C/C++ (Unreal Engine itself is C++), and a **native C/C++ rewrite of KEEL is a contemplated future** once it is fully implemented. This is not a worry and changes nothing now — it is *designed for*: the frozen golden cases are a **language-neutral conformance layer** (canon §7, ADR #5), so a future port simply re-passes the same goldens. By design the language is the most reversible decision in the system; the **contracts, not the language, are the longevity asset.**
